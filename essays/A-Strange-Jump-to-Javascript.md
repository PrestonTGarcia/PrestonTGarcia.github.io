Prior to learning Javascript as I have been in ICS 314, I thought it would be similar to other languages I have learned in the past, such as Java, Python, or C++; however, after doing the basic Javascript and ES6 modules on FreeCodeCamp, I have come to realize that Javascript is way different from any other programming language I have studied before, and can have way different syntax or functionalities than what I had done before.

Take for example, the arrow function in Javascript shown below:

```
const myFunc = () => {
    const myVar = “value”;
    return myVar;
}
```
Compared to the Python script below:
```
def myFunc():
  myVar = "value"
  return myVar
myFunc = myFunc()
```
In Javascript ES6, we can create anonymous functions and just assign the value of that anonymous function to a variable or constant. In any other language that I’ve learned in the past, we always needed to create a function with a name, and call on that function somewhere else in the code to set the return value to a variable, while in Javascript we can conveniently have an anonymous function set to a variable with the arrow operator. I believe that overall with my current experience in Javascript, it’s a much more convenient language than the others.

In class, one of the ways that we learn is through athletic software engineering, which means we would have to do time based software engineering workouts of the day(WODs). So far, we’ve done a few practice WODs and this way of practicing software engineering is stressful, yet I feel that it will be useful with what we plan to do in the future. Prior to this class, my programming projects were always given a far deadline, not a given amount of time to come up with an algorithm and code it. This time limit adds a bit of stress, and initially I thought it wouldn’t make sense to have this sort of time limit; however, in the real world, I know that there’s going to be times where we’re going to be crunched on time, and these practice WODs are great practices for when that time comes.

This semester will be a strange one when it comes to programming, as I’m in the process of learning a completely new language that’s different from anything that I’ve ever done before, and through a different style that I’ve done before, with WODs instead of deadlines. Through these, I’m hoping that I can improve myself not only in Javascript, but also my programming mindset.
