## The Importance of Questions

Questions are an essential part of not only the software engineering field, but in any sort of group, team, or organizational environment. We want our questions to be clear, concise, and smart in order to get an answer that fulfills what we are questioning. Going back to the software engineering field, questions are often asked about how to do certain tasks using a programming language, as there are often people who have either experienced the same problem that is happening, or other developers know a way to fix the problems we encounter.

After reading ["How To Ask Questions The Smart Way"](http://www.catb.org/esr/faqs/smart-questions.html) by Eric Raymond, I was able to identify what is a smart question, what isn't a smart question, and what the smartest questions are. Not-so-smart questions often lack the details needed to solve the problem, and don't make it clear what the problem is, and lead to more questions asked as a response, instead of a clear answer that "solves" the problem introduced. Smart and smartest questions include details to identify the problem, as well as other details that might lead to the solution to the questions such as what type of environment is being used, and what's already been tried before to solve the problem. To show what each degree of "smartness" in a question looks like, I explored [StackOverflow](https://stackoverflow.com/) and found an example for each.

## Not-So-Smart

To start off, I found a [not-so-smart question](https://stackoverflow.com/questions/60103844/please-help-guys) that has even been closed, due to it's lack of detail or clarity.

The header of this question already throws up red flags as a not-so-smart question, as it's undescriptive and meaningless. As shown above in the link to the post and in the screenshot, the header says "Please help, guys". This gives no detail on what the user needs help with, as a result it loses attention of people who could potentially help out with what the user is experiencing.

Aside from the header of the post, the user is very undescriptive in the content of their post as well. The user's first words in the post are "Its app / site like brain trainer". This could lead to confusion, as the user doesn't explain exactly what they want in their app / site, such as what part of it is like brain trainer? What IS brain trainer? Users reading questions shouldn't have to research these things or have to ask the original poster of these things, rather, explanations should be given as a part of the question. 

The user ends their question with a snippet of their code and "Please help, I need to add the button which submit the text in input and check,if it right". The English in this question is hard to understand, and there's no inclusion of errors or what has been done to try and do what is desired so far. The user seems to demand more of an open tutorial, rather than a clear answer.

## Smart

Next, I found a [smart](https://stackoverflow.com/questions/65879177/str-object-has-no-attribute-name-in-discord-py-issue) question about discord.py, and an issue that has been experienced by the user.

The user gives a good description of what they want to do, as they say "i needed a way to store a lot variables in one it worked using the class function but the variable(p0.name, p0.car etc) doesnt print and it doesnt allow me to change the value.", includes a snippet of their code, and what the error contains. The user also includes a list of previous actions that have been done to fix the error. This question is smart, as it contained enough details to have a reader respond with an answer. 

Although this question contains some smart aspects, it can be improved to be even smarter. The English of the post seems to be a little off, as there are a few grammatical errors with capitalization and punctuation. There's also a lack of explanation of what environment the user is running this on, such as what version of discord.py they're using, or what version of Python they're using. In this case, these didn't make too much of a difference, but they often will make a difference in how the code will be typed.

## Smartest

Lastly, I found a [smartest](https://stackoverflow.com/questions/26155776/importerror-no-module-names-matplotlib-python-3-3) question about importing matplotlib into Python 3.3.

The user right off the bat tells readers exactly what version of Python they are using in the header, Python 3.3. The user also explains what steps they took in attempting to solve the problem, crossing some potential solutions off and helping out readers. The user includes what types of errors they ran into, and even explains what OS environment they're using, Ubuntu 12.04. Readers were able to supply the user with an answer to solve the problem that was occuring with the user.

When asking questions, we should attempt to make it in the smartest way possible by following [Eric Raymond's essay](http://www.catb.org/esr/faqs/smart-questions.html). We should include details such as what exactly is it that we need, what environment we're running on, and what steps we've taken in attempting to solve the problem in a professional fashion with correct English or giving disclaimers that we may have language difficulties.
